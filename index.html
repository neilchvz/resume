Diligex — Internal Implementations & Automations

ISP Outage Monitoring & Ticket Automation
Rewst · UptimeRobot · ConnectWise Manage
  • Built a fully automated ISP outage ticket lifecycle in Rewst, eliminating all manual handling between circuit detection and ticket resolution. The workflow runs every minute and manages both ticket creation on outage and ticket closure on recovery without any technician involvement.
  • UptimeRobot monitors ISP circuits across client sites. When a circuit goes down Rewst detects the status change, identifies the correct client and board in ConnectWise, and opens a ticket automatically. When the circuit recovers Rewst locates the open ticket, posts an internal closure note, and closes it.
  • Replaced two previous failed approaches: a ConnectWise Automate ping monitor that silently failed whenever the assigned device was retired, and a ConnectWise Manage email parsing rule that required a senior engineer to update routing logic every time a new client or circuit was added. Now a Level 2 technician adds a monitor in UptimeRobot and the automation handles everything from the next run forward, no workflow changes, no senior involvement.
  • Handles multiple simultaneous outages correctly, avoids duplicate ticket creation, and only closes tickets when UptimeRobot confirms the circuit is genuinely back up.
  • Built as one main trigger workflow and two subworkflows, one handling ticket creation and one handling ticket closure with internal notes.
Throughput Monitoring & Alert Routing
Addigy · ConnectWise Manage · Proactive Monitoring
  • Designed and deployed proactive, site-aware throughput monitoring for client sites, catching internet degradation before clients notice it and replacing a fully reactive process where the client had to call first.
  • Limited monitoring to a single designated stationary office device per site using Addigy Flex Policy serial number targeting, removing result noise from laptops, mobile devices, and remote workers.
  • Calculated a custom alert threshold for each client based on their actual network setup: contracted ISP speed, firewall hardware ceiling, AMP/IDP overhead, and client VPN impact, instead of just applying one flat percentage to everyone.
  • The alert's subject line does the routing. ConnectWise reads it to identify the client, assign the right board, apply the SLA, and page on-call for 24x7 clients automatically. No manual handling between alert and ticket.
FileVault Compliance Automation
Addigy · macOS Security & MDM Automation
  • Built a Diligex-wide operational automation to eliminate manual FileVault compliance checks across all managed macOS fleets. Previously technicians had to manually verify encryption status and escrow recovery keys per device. This automation is continuous with no scheduled audits and no technician involvement required.
  • Built two continuous Addigy Flex Policies to detect and remediate FileVault compliance gaps, covering devices with FileVault disabled and devices that were encrypted before MDM enrollment with no recovery key escrowed.
  • Policy 1 targets devices with FileVault on but no key escrowed. It deploys Escrow Buddy (open-source, built by Netflix's Mac engineering team) to silently rotate and escrow the recovery key at the user's next login, then uninstalls itself automatically. No user-facing prompts.
  • Policy 2 targets devices with FileVault off entirely. It deploys an MDM profile that enables FileVault via deferred enablement and escrows the key to Addigy at the same time. Users get walked through a logout/login credential prompt and a branded IT notification explaining the change.
  • Both policies run continuously and indefinitely. Devices that fall out of compliance for any reason get detected and remediated automatically on the next Addigy check-in. No scheduled audits, no manual intervention, no maintenance overhead.
